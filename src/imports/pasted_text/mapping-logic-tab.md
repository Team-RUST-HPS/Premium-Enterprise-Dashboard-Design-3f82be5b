Design the **Mapping Logic tab view** for a data transformation platform.
This screen already exists within the product and must **reuse the existing UI theme, spacing, typography, and visual system**. Do not introduce a new design style. The goal is to improve the **mapping interaction and editing experience**.

The mapping tab allows users to view and modify mappings that were **automatically generated by the backend based on a Business Analyst specification**.

Users must be able to quickly review, remove, and modify these predefined mappings.

---

Mapping Workspace Layout

Use a **three-column layout** within the existing mapping tab.

Left panel
Source schema structure

Center panel
Mapping visualization canvas

Right panel
Target schema structure

Both source and target panels display **hierarchical structures**.

---

Schema Structure Display

The source and target schemas should appear as **structured trees with elements aligned vertically**.

Example source schema:

ISA
ISA06 Sender ID
ISA08 Receiver ID

GS
GS01 Functional ID

ST
ST01 Transaction Code

Example target schema:

sender_id
receiver_id
transaction_type

Each schema element should appear as a **compact row with a clear center anchor point**.

---

Mapping Connections

Mappings between source and target elements should be displayed using **very thin connector lines**.

Important rule:

Connector lines must originate from the **center point of the source element** and connect to the **center point of the target element**.

This improves connection detection and avoids visual confusion when many mappings exist.

Connector style:

thin line
subtle glow
minimal visual weight

Lines should appear automatically based on the predefined mapping generated from the BA specification.

---

Mapping Editing Controls

When a user selects a mapping line or schema element pair, a **mapping edit popover** should appear.

The popover should allow the user to:

Remove mapping
Modify transformation function
Add transformation function

Removing a mapping should instantly remove the connector line.

---

Transformation Function Selector

Provide a **function selector dropdown** for transformation logic.

The dropdown must include multiple function categories:

EDI transformation functions
JSON transformation functions
XML transformation functions

Examples:

trim()
substring()
uppercase()
lowercase()
concat()
dateFormat()
lookup()
split()
replace()

The dropdown options should **adapt based on source and target data properties**.

Example logic:

If source is string → show string functions
If source is numeric → show numeric functions
If target is date → show date functions

---

Function Suggestions

When a user selects a mapping, the system should display **function suggestions automatically**.

Example suggestion panel:

Suggested transformations:

trim()
uppercase()
substring(0,10)

Users can click a suggestion to apply it instantly.

Suggestions should be based on:

source data type
target data type
common mapping patterns

---

Transformation Logic Display

When a transformation function is applied, show a **small logic box on the connector line**.

Example:

ISA06 → trim() → sender_id

Logic boxes should be compact and easily editable.

Clicking the logic box reopens the function selector.

---

Handling Large Mapping Sets

The interface must support **very large mapping sets**.

Connector lines should remain visually clean even when thousands of mappings exist.

Inactive mappings should appear slightly faded.

Selected mappings should highlight clearly.

---

Mapping Status Indicators

Schema elements should visually indicate mapping state.

States:

Mapped
Unmapped
Mapped with transformation

Example indicators:

small icon next to element name
subtle highlight

---

Filtering Tools

Add filtering options to help navigate large schemas.

Filters:

Show unmapped fields
Show mapped fields
Show transformed fields

Provide a search box for quickly locating schema elements.

---

User Experience Goals

The mapping interface should allow users to:

Quickly review predefined mappings from BA specifications
Remove incorrect mappings
Apply transformations easily
Understand relationships between schema elements clearly

Even with large schemas, the interface must remain **fast, readable, and easy to navigate**.
